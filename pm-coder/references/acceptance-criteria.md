# 验收标准参考（pm-coder）

> 供 pm-coder 子Agent在交付前做自检的验收标准清单。需要时通过 read_file 渐进加载。

## 通用验收标准

所有编码任务的交付物必须满足以下标准：

### 代码质量
- [ ] 无 TypeScript/Python 类型错误
- [ ] 无 ESLint/Pylint 警告（或已有合理 suppress）
- [ ] 无 `console.log` / `print` 调试代码残留
- [ ] 无硬编码敏感信息（密钥、密码、token）
- [ ] 无 `any` 类型（TS）或裸 `except:`（Python）

### 错误处理
- [ ] 所有异步操作有 try-catch
- [ ] 错误信息对用户友好（非技术术语）
- [ ] 关键操作有日志记录

### 测试
- [ ] 核心功能有单元测试
- [ ] 测试覆盖正常路径 + 主要异常路径
- [ ] 测试可独立运行（无外部依赖）

### 文档
- [ ] 复杂函数有 JSDoc / Pydoc
- [ ] 公开 API 有使用说明
- [ ] 文件头注释说明用途

## 分类验收标准

### 前端开发（Vue3/React）
- [ ] 组件语义化命名
- [ ] Props 有类型定义和默认值
- [ ] 响应式布局（移动端适配）
- [ ] 无内存泄漏（定时器/事件监听器清理）
- [ ] 加载状态和空状态处理

### 后端开发（FastAPI/Express）
- [ ] API 端点有请求参数校验
- [ ] 返回标准格式 `{ code, message, data }`
- [ ] 数据库操作有事务保护
- [ ] 敏感接口有权限校验

### 数据库
- [ ] 表结构有索引设计
- [ ] 有外键约束或应用层约束
- [ ] 迁移脚本可重复执行（幂等）

### 调试修复
- [ ] 提供最小复现步骤
- [ ] 说明根因分析
- [ ] 修复后有回归测试
